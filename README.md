# VFD 显示器

基于 ESP8266 的 VFD 显示屏。

## 硬件

- **主控**: ESP8266 (ESP-12E)
- **显示屏**: VFD 8-MD-06INKM (8 位数字, 每位 5×7 点阵)
- **RTC**: SD3078 (带可充电电池，上电自动充电，掉电保持时间)
- **I/O 扩展**: PCF8574 (控制高压/灯丝使能, 读取 4 个按键，LED)
- **蜂鸣器**: 无源蜂鸣器

### 硬件版本

项目支持两个硬件版本（0.2.x 和 0.3.x），在 `src/config.h` 中通过宏切换。请在构建之前确认好 PCB 上的版本并定义对应的宏，再进行编译。

## 功能说明

### 按钮说明

| 按键       | 功能描述                             |
|------------|--------------------------------------|
| 模式键     | 短按切换显示模式，长按重启设备     |
| 时间同步键 | 短按无功能，长按手动触发 NTP 校时 |
| 亮度+键    | 短按增加亮度，长按重置设备（WiFi 配置清除） |
| 亮度-键    | 短按降低亮度 |

0.3.x 的按键标识在 PCB 背面；0.2.x 的按键没有标识，从屏幕面看，按键从左到右依次为：亮度-键、亮度+键、时间同步键、模式键。

### 三种显示模式

通过短按模式键循环切换:

1. **Clock** — 从 SD3078 RTC 读取时间, 显示 `HH:MM:SS`
2. **Serial** — 接收串口文本, 遇到 `\n` 后显示最近 8 个字符 (不足 8 位右对齐)
3. **AIDA64** — 解析 AIDA64 Remote Sensor 串口协议, 显示 PC 监控数据

### 滚动动画

时间切换时支持逐位滚动动画。

### 亮度控制

短按亮度+/-键, 调节 8 级亮度, 并自动保存。

### WiFi / NTP

- 启动自动尝试连接上次配置的 WiFi，如果连接失败则进入 WiFi 配网模式
- 连接成功后自动 NTP 校时
- 长按 NTP 同步键手动校时
- 长按亮度+键清除配置并重启

### 配网

配网模式下 ESP8266 会始终开启一个开放 WiFi 热点，热点名称为 "ESP-XXXXXX"（其中 XXXXXX 是芯片 ID 的后 6 位）。

连接到该热点后，一般情况下会自动弹出配置页面，按 `Configure WiFi` 按钮进入 WiFi 列表，选择目标 WiFi 并输入密码后提交，设备会自动重启并连接到新的 WiFi 网络。

如果没有自动弹出配置页面，可以手动访问 `http://192.168.4.1` 进行配置。

如果需要重置 WiFi 配置，可以访问它的 IP 地址并在页面上重新配置，也可以直接长按亮度+键，设备会清除保存的 WiFi 信息并重启，进入配网模式。


## 项目结构

```
src/
├── main.cpp              # 主循环, 按键处理, 模式分发
├── config.h              # 引脚定义, 按键映射 (硬件版本选择)
├── Font5x7.h             # 5×7 点阵字库
├── DisplayManager.h/cpp  # VFD 显示管理, 滚动动画
├── ButtonHandler.h/cpp   # 按键消抖与长短按检测
├── ModeManager.h/cpp     # 模式状态机
├── RTCManager.h/cpp      # SD3078 RTC + NTP 客户端
├── SerialManager.h/cpp   # 串口文本接收与显示
├── AIDA64Handler.h/cpp   # AIDA64 协议解析
├── PCF8574Handler.h/cpp  # PCF8574 I/O 扩展器封装
lib/
├── VFD_8MD06INKM/        # VFD 显示屏驱动库 (自定义)
└── SD3078/               # SD3078 RTC 驱动库 (自定义)
```

## 构建与烧录

使用 PlatformIO 构建并烧录。克隆项目后，按硬件版本修改 `src/config.h` 中的宏定义，然后执行:

```bash
pio run -t upload
```

监视串口:

```bash
pio device monitor -b 9600
```
